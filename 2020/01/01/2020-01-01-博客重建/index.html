博客重建

前言

习惯了在脑子里记忆（当然只是脑子说会了），很少使用纸笔，博客也搁置了很长时期，但久而久之，当再次遇到一些有意思的、或者值得去记忆去练习的点总感觉似曾相识却又如若初见。2020年的到了着实让我对未来有了些许的担忧，故重建博客，一切从零开始，记录下自己学习中遇到的知识点，以及学习笔记等，用于今后巩固。

博客的建立

搭建博客还是使用了以前学长推荐的github+hexo，搭建起来十分简单，具体步骤什么的网上教程很多，在这也不用叙述。以前更新文章总在本地上编写后直接hexo g -d完事，可是有次在电脑不得已重置后，本地文件没了，在github仓库中也只有public文件夹中的文件，无法继续更新文章，只能重新搭建环境。偶然在网上看到一种方法，是将hexo在本地文件夹上传到github上的另一个分支上（假设分支名hexo），更新文章后将他提交到hexo分支，最后再执行hexo g -d即可完成文章更新，如果需要使用另外一台电脑更新文章，只需要将hexo分支上的文件pull下来到本地，当然git、node等必须要先安装。

因为github上是开源的，所以当将本地文件上传到hexo分支时，所有的源文件是对所有人可见的（当然也可以直接在github中设置成私有库，但是付费我就多绕了点路），所以以下操作是将本地文件托管至coding中，因为coding中可以免费设置私有库。

在coding中设置好一个私有库，并将本地文件上传到这个库中，除了node_modules、public、.deploy_git,如果之前有更新或者备份过主题，那么还要排除掉主题文件下的.git文件，因为要是不删除，那么在另外电脑上会没有办法运行（有两个git库）。

文章上传

像以前一样，在写完文章后保存到/source/_post文件夹下

​ 1.git add .或者git add xxx.md两种方式均可(xxx表示需要提交的文件名，.表示所有文件)

​ 2.git commit -m "xxxx"（xxxx表示此次提交的注释）

​ 3.git push这一步是将文件提交到默认分支（coding中的默认分支master）上去（需要登陆）

以上三步操作后，文件将更新到默认分支上，但查看博客后发现还没有更新，因为还差一步。

​ 4.执行hexo g -d

只需上述一步，文章即可正常的更新了。此时就完成了两个地方的更新，coding中源文件的更新以及github上博客的更新。

git pull 将库中内容拉取下来，一般进行的是将本地文件更新与库中文件同版本。
